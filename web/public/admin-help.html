GuildPilot Admin Command Guide

Live command reference aligned with current module-first architecture.

Compatibility Notes
  • Canonical public API is /api/public/v1/*. Legacy /api/public/* aliases remain for compatibility.
  • Legacy governance aliases /propose, /support, /vote are still available.
  • /og-config is deprecated; use /verification admin og-*.
  • /governance propose and the portal proposal form both support optional category and cost/costIndication fields.

🔐 Verification

  • /verification status — Show verification status, tier, VP, holdings.
  • /verification wallets — List linked wallets.
  • /verification refresh — Re-scan holdings and refresh user data.
  • /verification quick — Start micro-transfer quick verification.

Admin: /verification admin panel, export-user, remove-user, export-wallets, token-role-add, token-role-remove, token-role-list, role-config (tier config is portal-managed), actions, og-view, og-enable, og-role, og-limit, og-sync, activity-watch-add, activity-watch-remove, activity-watch-list, activity-feed, activity-alerts.

🏛️ Governance

  • /governance propose — Create proposal. Optional category and cost fields are supported.
  • /governance support — Support draft proposal.
  • /governance vote — Vote yes/no/abstain.
  • /propose, /support, /vote — legacy aliases (still supported).

Admin: /governance admin list, cancel, settings.

💰 Treasury

  • /treasury view — Public-safe treasury snapshot.

Admin: /treasury admin status, refresh, enable, disable, set-wallet, set-interval, tx-history, tx-alerts.

⚔️ Battle

  • /battle create — Create battle lobby with optional role gating and max players. (Admins/Mods only)
  • /battle start — Start your open lobby (min players required). (Admins/Mods only)
  • /battle cancel — Cancel your open lobby.
  • /battle stats — Show battle stats and leaderboards.

Role Gating (Optional)

When creating a lobby, you can control who can join:

  • Required Roles (ANY): required_role_1, required_role_2, required_role_3 — User must have at least one of these roles to join.
  • Excluded Roles (NONE): excluded_role_1, excluded_role_2, excluded_role_3 — User cannot have any of these roles; they will be auto-removed if they react to join.

Example: Require @Member or @Supporter, exclude @Bot and @Muted:
/battle create max_players:10 required_role_1:@Member required_role_2:@Supporter excluded_role_1:@Bot excluded_role_2:@Muted

Channel Rule: only one active battle per channel at a time (open or in-progress).

Closure Behavior: cancel/force-end updates the original lobby message and removes reactions so the battle is clearly closed.

Admin: /battle admin list, force-end, settings.

🎯 Heist

Module is currently disabled by default. Commands exist for future re-enable:

/heist view, /heist signup, /heist status, admin: create, list, cancel.

⚙️ Config

  • /config modules — View module toggles.
  • /config toggle — Enable/disable module.
  • /config status — Runtime status overview.

🖥️ Web-Only Admin Surfaces

These are managed in /admin and intentionally do not have full slash-command parity:

  • Ticketing categories/panel/archive controls
  • Analytics dashboard
  • Self-Serve Roles UI
  • Voting Power mapping table
