💼 Wallet Setup Guide

How to Link a Wallet

  1. Click the "Add Wallet" button in the Wallets section
  2. Choose Sign Message (free) or Micro Transaction
  3. Your wallet extension (Phantom, Solflare, etc.) opens automatically
  4. Approve the signature or transaction in your wallet
  5. Done — your wallet address is detected automatically, no copy/paste needed

Supported Wallets

  • Phantom - Popular browser extension and mobile app
  • Solflare - Web-based and mobile wallet
  • Backpack - Modern multi-chain wallet
  • Any Solana wallet that supports message signing

Setting a Primary Wallet

Click the "Set Primary" button next to any verified wallet. This wallet will be used for primary voting power calculations and NFT verification.

Security Best Practices

  • Never share your private key or seed phrase
  • Only sign messages you understand
  • Verify you're on the official portal before signing
  • Keep your wallet software up to date

📜 Governance Guide

Creating a Proposal

  1. Click Create Proposal in the Governance section, or use /governance propose in Discord
  2. Provide a clear, descriptive title for your proposal
  3. Write a detailed description explaining the proposal's purpose and impact
  4. Optionally choose a category and add a cost indication for context
  5. Wait for admin review and approval before voting begins
  6. Engage with the community to gather support

Proposal Lifecycle

  • Draft - Proposal created, awaiting admin approval
  • Supporting - Gathering community support before voting
  • Voting - Active voting period, cast your vote
  • Passed - Proposal approved by the community
  • Rejected - Proposal did not meet approval threshold
  • Closed - Voting period ended

Voting on Proposals

  1. View active proposals in the Governance section
  2. Read the proposal details carefully
  3. Click Yes, No, or Abstain on any active proposal, or use /governance vote in Discord
  4. Your voting power is automatically calculated based on your tier
  5. Track voting progress and results in real-time

⚔️ Battle Lobbies

Creating a Battle Lobby

Use the command: /battle create (Admins/Moderators only)

Optional Parameters:

  • max_players — Maximum number of participants (default: 999)
  • required_role_1/2/3 — Require at least ONE of these roles to join
  • excluded_role_1/2/3 — Block users with ANY of these roles

Channel Rule: only one active battle per channel at a time.

Role Gating Explained

Required Roles (ANY Logic): A user must have at least one of the specified roles. If you set required_role_1:@Member and required_role_2:@Supporter, any user with @Member OR @Supporter can join.

Excluded Roles (NONE Logic): A user cannot have any of the specified roles. If you set excluded_role_1:@Bot and excluded_role_2:@Muted, users with @Bot OR @Muted are blocked.

Example: Members-Only Battle

Create a battle for members and supporters only, excluding bots:

/battle create max_players:20 required_role_1:@Member required_role_2:@Supporter excluded_role_1:@Bot

Joining a Battle

  1. React with ⚔️ to a battle lobby message to join
  2. If you meet role requirements, you're added to the participant list
  3. If you don't meet requirements, your reaction is removed and you'll receive a DM explaining why
  4. An admin/mod starts the lobby with /battle start once minimum players are reached

Battle Commands

  • /battle start — Start your open lobby (requires minimum players, Admin/Mod only)
  • /battle cancel — Cancel your open lobby (closes lobby message and removes reactions)
  • /battle stats — View battle statistics and leaderboards

Elite Four Phase

  • Automatically triggers when 4 fighters remain
  • All remaining fighters reset to 100 HP
  • No lucky escapes / revivals
  • Higher combat intensity and dramatic narration

🔧 Troubleshooting

Wallet Verification Failed

Issue: Signature verification returns an error

Solutions:

  • Make sure your wallet extension is unlocked before clicking the verify button
  • Try a different wallet application (switch between Phantom, Solflare, etc.)
  • Clear your browser cache and cookies, then try again
  • Make sure your wallet software is up to date
  • If using a hardware wallet, ensure it's connected and the Solana app is open

Tier Not Updating

Issue: Your displayed tier doesn't reflect your current NFT count

Solutions:

  • Run /verification refresh in Discord to force a holdings refresh
  • Wait 2-3 minutes for the blockchain verification to complete
  • Ensure your NFTs are in a wallet that's been verified in the system
  • Check that you've set your primary wallet correctly
  • Contact support if the issue persists after 5 minutes

Cannot See Admin Features

Issue: Admin panel is not visible in the navigation

Solutions:

  • Verify you have administrator role in the Discord server
  • Log out completely and log back in to refresh permissions
  • Clear browser cache and session storage
  • Contact a server administrator if you should have access

Voting Not Working

Issue: Unable to cast votes on proposals

Solutions:

  • Ensure you have at least 1 verified NFT (minimum tier requirement)
  • Check that the proposal is in "Voting" status
  • Verify you haven't already voted on this proposal
  • Try refreshing the page and voting again, or use /governance vote in Discord

Still Need Help?

If your issue isn't resolved by the solutions above, please reach out to support:

  • Visit the #support channel in Discord
  • Provide detailed information about your issue
  • Include screenshots if applicable
  • Mention any error messages you received
